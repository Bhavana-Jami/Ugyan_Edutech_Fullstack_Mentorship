üìù1. Responsive Text

Goal:

Make text adapt to screen size without breaking layout

Techniques:

  • Use relative units like em, rem, vw, % instead of px
  • Use media queries to change font size on different screens
  • Use clamp() for fluid typography
h1 { font-size: clamp(1.5rem, 5vw, 3rem); /* min, preferred, max */ } @media (max-width: 768px) { p { font-size: 1rem; } }

üñºÔ∏è2. Responsive Images

Goal:

Ensure images resize without overflow and are sharp on all devices

Techniques:

  • Use max-width: 100%; height: auto;
  • Use <picture> and sizes for responsive image loading
  • Wrap images inside a container and make the container responsive
img { max-width: 100%; height: auto; } .image-container { width: 100%; max-width: 600px; margin: 0 auto; }
<picture> <source media="(max-width: 768px)" srcset="mobile.jpg"> <source media="(max-width: 1200px)" srcset="tablet.jpg"> <img src="desktop.jpg" alt="Responsive image"> </picture>

üé¥3. Responsive Groups of Elements

Goal:

Align items like cards, team members, products responsively

Techniques:

  • Use Flexbox or CSS Grid with wrap or auto-fit/minmax
  • Use media queries to change layout direction or number of columns
  • Implement flexible container sizing
/* Flexbox Approach */ .container { display: flex; flex-wrap: wrap; gap: 1rem; } .card { flex: 1 1 300px; } /* CSS Grid Approach */ .grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); gap: 1rem; }

üß≠4. Responsive Navigation

Goal:

Navigation should adapt to smaller screens (hamburger menu or stack layout)

Techniques:

  • Use Flexbox or Grid for flexible layouts
  • Use media queries to stack or hide/show menu items
  • Implement JavaScript for toggle menus on smaller screens
  • Consider hamburger menu patterns for mobile
.navbar { display: flex; justify-content: space-between; align-items: center; } .nav-links { display: flex; gap: 2rem; } @media (max-width: 768px) { .nav-links { display: none; flex-direction: column; position: absolute; top: 100%; left: 0; width: 100%; background: white; } .nav-links.active { display: flex; } }

üìã5. Responsive Forms

Goal:

Make form inputs and layout adjust to screen size

Techniques:

  • Use width: 100% or flex/grid for input fields
  • Group fields with responsive containers
  • Stack labels and inputs on smaller screens
  • Ensure touch-friendly button sizes
.form-group { display: flex; flex-direction: column; margin-bottom: 1rem; } .form-input { width: 100%; padding: 0.75rem; border: 1px solid #ddd; border-radius: 4px; } @media (min-width: 768px) { .form-row { display: flex; gap: 1rem; } .form-group { flex: 1; } }

üìä6. Responsive Tables

Goal:

Tables should scroll or stack nicely on smaller screens

Techniques:

  • Add overflow-x: auto to parent container
  • Use CSS to turn table into stacked blocks
  • Implement card-based layouts for mobile
  • Hide non-essential columns on smaller screens
.table-wrapper { overflow-x: auto; } @media (max-width: 768px) { .responsive-table { display: block; } .responsive-table thead { display: none; } .responsive-table tr { display: block; margin-bottom: 1rem; border: 1px solid #ddd; border-radius: 8px; } .responsive-table td { display: block; padding: 0.5rem; text-align: right; } .responsive-table td:before { content: attr(data-label) ": "; font-weight: bold; float: left; } }

üìè7. Responsive Spacing & Margins

Goal:

Avoid fixed gaps that break on smaller screens

Techniques:

  • Use em, rem, %, or clamp() for margin/padding
  • Avoid fixed pixel values like margin-left: 200px
  • Use CSS custom properties for consistent spacing
  • Implement a spacing scale system
:root { --spacing-xs: clamp(0.5rem, 2vw, 1rem); --spacing-sm: clamp(1rem, 3vw, 1.5rem); --spacing-md: clamp(1.5rem, 4vw, 2rem); --spacing-lg: clamp(2rem, 5vw, 3rem); } .section { padding: var(--spacing-md); margin-bottom: var(--spacing-lg); } .container { max-width: 1200px; margin: 0 auto; padding: 0 clamp(1rem, 5vw, 2rem); }

üé¨8. Responsive Media (Videos, Iframes)

Goal:

Videos and iframes should not overflow container

Techniques:

  • Wrap in a container with aspect ratio
  • Use intrinsic ratio technique
  • Implement CSS aspect-ratio property
  • Make embedded content fluid
.video-container { position: relative; padding-bottom: 56.25%; /* 16:9 aspect ratio */ height: 0; overflow: hidden; } .video-container iframe { position: absolute; top: 0; left: 0; width: 100%; height: 100%; } /* Modern approach using aspect-ratio */ .video-modern { width: 100%; aspect-ratio: 16 / 9; }

Goal:

Use utility classes for quick responsive adjustments

Techniques:

  • Use frameworks like Tailwind CSS
  • Create your own utility classes
  • Implement breakpoint-specific utilities
  • Use classes for common responsive patterns
/* Custom Utility Classes */ .hidden { display: none; } .block { display: block; } .flex { display: flex; } .grid { display: grid; } .w-full { width: 100%; } .w-half { width: 50%; } .w-quarter { width: 25%; } @media (min-width: 768px) { .md\\:w-half { width: 50%; } .md\\:block { display: block; } .md\\:hidden { display: none; } } @media (min-width: 1024px) { .lg\\:w-quarter { width: 25%; } .lg\\:grid { display: grid; } }

Pro Tip: Combine utility classes with custom CSS for maximum flexibility and maintainability!
