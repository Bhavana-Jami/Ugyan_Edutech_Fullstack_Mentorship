‚ö†Ô∏è Problems with Vanilla JavaScript

1. Manual DOM Manipulation

‚ùå Vanilla JS
// Manual and Error-Prone
const button = document.getElementById('myButton');
const counter = document.getElementById('counter');
let count = 0;

button.addEventListener('click', () => {
    count++;
    counter.textContent = count; // Manual DOM update
    // What if we forget to update other elements showing count?
});

2. State Management Chaos

‚ùå Scattered State
// Scattered state everywhere
let userName = '';
let isLoggedIn = false;
let cartItems = [];

// Multiple update functions
function updateUserName(name) {
    userName = name;
    document.getElementById('username1').textContent = name;
    document.getElementById('username2').textContent = name;
    // Easy to miss updating all related UI!
}

3. Other Issues:

  • No reusability - Copy-paste code everywhere
  • Poor scalability - Hard to maintain large apps
  • Memory leaks - Forgotten event listeners
  • No structure - Code becomes spaghetti

‚öõÔ∏è Why ReactJS?

React Solution: React provides organized, predictable, and reusable component architecture with automatic UI updates.

  • Solves DOM complexity with Virtual DOM
  • Component-based: Reuse logic/UI blocks
  • Unidirectional data flow (predictable)
  • Great developer experience
  • Active community and ecosystem

üéØ ReactJS Core Fundamentals

1. JSX (JavaScript XML)

  • Syntax extension: HTML-like code in JS
  • Must return a single parent element
  • Use {} for JS expressions inside JSX
  • Not HTML, it gets compiled to React.createElement()

2. Functional Components

‚úÖ React Component
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}
  • Just JS functions returning JSX
  • Easy to read, reusable

3. Props (Properties)

  • Passed from parent to child component
  • Example: <Welcome name="Bhavana" />

4. useState Hook (State)

‚úÖ State Management
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}
  • Manages internal state
  • Causes re-render when state updates

üèóÔ∏è Thinking in Components & Parent-Child Relations

Component Hierarchy Example:

App (Parent)
‚îú‚îÄ‚îÄ Header (Child)
‚îú‚îÄ‚îÄ UserProfile (Child)
‚îÇ ‚îú‚îÄ‚îÄ Avatar (Grandchild)
‚îÇ ‚îî‚îÄ‚îÄ UserInfo (Grandchild)
‚îî‚îÄ‚îÄ Footer (Child)

Data Flow Rules:

  • Top-down: Data flows from parent to child via props
  • Unidirectional: Child cannot directly modify parent state
  • Communication: Child talks to parent via callback functions
Parent Component
function App() {
  const [user, setUser] = useState("John");
  
  return (
    <div>
      <UserProfile 
        name={user} 
        onNameChange={setUser} 
      />
    </div>
  );
}
Child Component
function UserProfile({ name, onNameChange }) {
  return (
    <div>
      <h2>{name}</h2>
      <button onClick={() => onNameChange("Jane")}>
        Change Name
      </button>
    </div>
  );
}
